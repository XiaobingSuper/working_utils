MoE FLOPs and Bandwidth Calculator

Computes MoE GEMM1/GEMM2 FLOPs from routed rows (M * topk) and estimates lower-bound memory traffic for dtype/scale choices.

Shape and Timing

Weight traffic uses "experts read for weights", capped by min(E, M * topk). For balanced/random large-M MoE this is usually close to E.

DType Byte Presets

gemm2 output traffic modes: write per topk route is the GEMM-only default and counts partial output [M, topk, model_dim]; final output write once only counts the final reduced [M, model_dim] output lower bound.

GPU Theoretical Roofline

Presets are for quick roofline comparison only. MI355X uses AMD-published FP8 5 PFLOPS and 8 TB/s HBM. MI300X uses AMD-published FP8 2614.9 TFLOPS and 5.325 TB/s HBM, and is used here as the MI308 fallback. MI400 uses AMD preview numbers.

Variable Guide

Symbol Input Field Meaning
M M / tokens Number of input tokens routed through MoE.
model_dim / K model_dim / K Hidden size. It is the GEMM K dimension for gemm1 and output N dimension for gemm2.
inter_dim / I inter_dim / I Expert intermediate size after activation. For SwiGLU gemm1 computes gate and up, so gemm1 N is 2 * I.
E experts / E Total number of experts in the layer.
topk / T topk / T Experts selected per token. MoE compute uses routed rows M * topk, not M * E. The calculator caps effective topk at E.
active_E experts read for weights How many unique experts' weights are read at least once. The calculator caps this at min(E, M * topk); enter a lower measured value for skewed routing.
time_us gemm1 time, gemm2 time Measured kernel time in microseconds. TFLOPS and BW are divided by this value.
out traffic mode gemm2 output traffic Controls gemm2 output traffic: GEMM-only partial output per route, or final reduced output lower-bound.
*_B DType byte presets Bytes per element for each tensor. Example: fp8 is 1 byte, bf16 is 2 bytes, fp32 is 4 bytes.
group scale group size Number of values covered by one block scale. MXFP8/MXFP4 commonly use group size 32.
scale_B scale dtype Bytes per block-scale value. For e8m0 scale this is 1 byte.
The bandwidth model is a lower-bound traffic estimate. It counts the required activation, weight, output, scale, and topk-weight traffic, but it does not include routing metadata, cache misses, repeated tile loads, or temporary scratch traffic.

FLOPs Formula

effective_topk = min(topk, E)

gemm1_flops = 2 * M * effective_topk * model_dim * (2 * inter_dim)
gemm2_flops = 2 * M * effective_topk * inter_dim * model_dim
total_flops = gemm1_flops + gemm2_flops

TFLOPS = flops / (time_us * 1e-6) / 1e12
The leading 2 is for FMA as two operations: multiply plus add. Expert count does not appear in FLOPs because each token only computes its selected topk experts.

Traffic Formula

effective_topk = min(topk, E)
effective_active_E = min(active_E input, E, M * effective_topk)

gemm1_bytes =
  M * effective_topk * model_dim * A1_B
+ M * effective_topk * ceil(model_dim / group) * scale_B
+ effective_active_E * (2 * inter_dim) * model_dim * W1_B
+ effective_active_E * (2 * inter_dim) * ceil(model_dim / group) * scale_B
+ M * effective_topk * inter_dim * stage1_out_B
+ M * effective_topk * ceil(inter_dim / group) * scale_B

gemm2_bytes =
  M * effective_topk * inter_dim * A2_B
+ M * effective_topk * ceil(inter_dim / group) * scale_B
+ effective_active_E * model_dim * inter_dim * W2_B
+ effective_active_E * model_dim * ceil(inter_dim / group) * scale_B
+ gemm2_output_bytes
+ M * effective_topk * topk_weight_B
gemm2_output_bytes is configurable. The default GEMM-only mode uses M * effective_topk * model_dim * out_B. The final-output lower-bound mode uses M * model_dim * out_B.
